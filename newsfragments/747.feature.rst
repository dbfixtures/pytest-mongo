Add authentication support to ``mongo_noproc`` and ``mongo_proc``: both factories now accept ``username``, ``password``, ``auth_source``, and ``tls`` parameters. ``mongo_noproc`` additionally accepts a full ``uri``. When ``username`` is provided to ``mongo_proc``, mongod is started with ``--auth`` and the user is created via the localhost exception. All parameters can also be set via ini options (``mongo_username``, ``mongo_password``, ``mongo_auth_source``, ``mongo_uri``, ``mongo_tls``) or the corresponding CLI flags.
