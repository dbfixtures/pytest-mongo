Add authentication support to ``mongo_proc`` and ``mongo_noproc`` via ``username``, ``password``, and ``auth_source``. ``mongo_noproc`` additionally accepts ``tls`` and a full ``uri``. When ``username`` is provided to ``mongo_proc``, ``mongod`` is started with ``--auth`` and the user is created via the localhost exception. Matching ini options and CLI flags are available where supported.
